# SW-05: Watchdog Daemon

## Context
You are building a Go CLI called `agent-swarm`. Read SPEC.md for full spec.
This ticket depends on sw-01 (config/tracker), sw-02 (backend), sw-03 (dispatcher).
Those packages exist in internal/ — use their public APIs.

## Your Scope
Build the watchdog loop in `internal/watchdog/` and `cmd/watch.go`.

### 1. Watchdog Package (internal/watchdog/watchdog.go)
```go
type Watchdog struct {
    config     *config.Config
    tracker    *tracker.Tracker
    dispatcher *dispatcher.Dispatcher
    backend    backend.AgentBackend
    worktree   *worktree.Manager
    notifier   notify.Notifier
    events     *EventLog  // append-only event log
}

func New(cfg, tracker, dispatcher, backend, worktree, notifier) *Watchdog
func (w *Watchdog) RunOnce(ctx context.Context) error   // single pass
func (w *Watchdog) Run(ctx context.Context) error        // loop with interval
func (w *Watchdog) SpawnTicket(ctx context.Context, ticketID string) error  // create worktree + launch agent
```

### 2. RunOnce Logic (per SPEC.md watchdog loop)
```
1. List running agents via backend
2. For each running ticket in tracker:
   a. Agent exited + has commits? → dispatcher.MarkDone(ticket, sha) → spawn next unblocked
   b. Agent exited + no commits (attempt 1)? → auto-respawn
   c. Agent exited + no commits (attempt 2+)? → dispatcher.MarkFailed(ticket), notify
   d. Agent alive > max_runtime? → notify "stuck?"
3. If 0 running + spawnable tickets + RAM OK → idle auto-spawn (up to maxAgents)
4. Phase gate reached? → notify, stop spawning
```

### 3. SpawnTicket Flow
1. Check worktree exists, create if not
2. Copy prompt file to worktree as `.codex-prompt.md`
3. Append prompt-footer.md if it exists
4. Call backend.Spawn()
5. Update tracker status to "running"

### 4. Event Log
- Append events to `events.jsonl` (one JSON object per line)
- Event types: `ticket_spawned`, `ticket_done`, `ticket_failed`, `phase_gate`, `respawn`, `idle_spawn`
- Each event: `{type, ticket, timestamp, data}`

### 5. CLI Command
- `cmd/watch.go` — `swarm watch [--interval 5m] [--once] [--dry-run]`
- `--once`: single pass then exit
- `--dry-run`: evaluate and print what would happen, don't execute
