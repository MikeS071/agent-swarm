# SW-01: Scaffold + Config + Tracker

## Context
You are building a Go CLI called `agent-swarm` — a multi-agent development orchestrator.
Read SPEC.md in the repo root for the full specification.

## Your Scope
Build the foundation that ALL other packages depend on:

### 1. CLI Scaffold (cmd/)
- `cmd/root.go` — Cobra root command with `--config` flag (default `swarm.toml`)
- `cmd/init.go` — `swarm init <project>` scaffolds swarm.toml + tracker.json + prompts/
- `cmd/status.go` — `swarm status [--project X] [--json]` reads tracker, prints table
- `cmd/add_ticket.go` — `swarm add-ticket <id> --deps a,b --phase 2 --desc "..."`
- `main.go` at repo root

### 2. Config Package (internal/config/)
- Parse `swarm.toml` using `github.com/pelletier/go-toml/v2`
- Struct must match SPEC.md `swarm.toml` format exactly
- `Load(path string) (*Config, error)`
- `Default() *Config` for `swarm init`

### 3. Tracker Package (internal/tracker/)
- `Load(path string) (*Tracker, error)`
- `Save(path string) error`
- `Ticket` struct matching SPEC.md format (status, phase, depends, branch, desc, sha, startedAt, finishedAt)
- `SetStatus(id, status string) error`
- `GetSpawnable() []string` — returns ticket IDs where: status=todo, all deps are done, phase <= current active phase
- `GetByPhase(phase int) []Ticket`
- `ActivePhase() int` — lowest phase with non-done tickets
- `Stats() Stats` — done/running/todo/failed/blocked/total counts
- `DependencyOrder() []string` — topological sort for integration merge order

### 4. Sysinfo Package (internal/sysinfo/)
- `AvailableRAM() (int, error)` — MB, reads /proc/meminfo or uses syscall
- `CanSpawn(minRAM int) bool`

## Tests
Write tests FIRST for:
- Config parsing (valid toml, missing fields, defaults)
- Tracker CRUD (load, save, set status, get spawnable, dep resolution, topo sort)
- Sysinfo (mock /proc/meminfo)
