# SW-10: Add `agent-swarm archive` command to clean done items

## Problem
When a project completes many tickets, the tracker fills with "done" items. `status --watch` shows all of them, making it hard to see active work. Every refresh iterates all tickets including done ones.

## Scope
Add `agent-swarm archive` command that moves completed tickets out of the active tracker.

### `archive` subcommand
```bash
agent-swarm archive              # archive all done tickets
agent-swarm archive --phase 1    # archive done tickets from phase 1 only
agent-swarm archive --dry-run    # show what would be archived
agent-swarm archive --restore    # restore archived tickets back to tracker
```

### Implementation
1. Add `swarm/archive.json` file — same format as tracker tickets section but for archived items
2. `archive` command:
   - Read tracker, find all tickets with `status == "done"`
   - Move them to `swarm/archive.json` (append, don't overwrite)
   - Remove from `swarm/tracker.json`
   - Print summary: "Archived N tickets (phase X: Y, phase Z: W)"
3. `archive --restore`:
   - Read archive.json, move all back to tracker.json
   - Clear archive.json
4. Status table and TUI should only show active tracker tickets (archived ones hidden)
5. `status --all` flag to include archived tickets in display

### Files
- `cmd/archive.go` — new cobra subcommand
- `internal/tracker/archive.go` — archive/restore logic
