# SW-07: HTTP API + SSE Server

## Context
You are building a Go CLI called `agent-swarm`. Read SPEC.md for full spec.
Depends on sw-01, sw-03, sw-05.

## Your Scope
Build `swarm serve` — HTTP API with SSE event streaming in `internal/server/`.

### 1. Server Package (internal/server/)
All endpoints per SPEC.md "HTTP API — swarm serve" section. Implement every endpoint listed.

### 2. Event Bus (internal/server/events.go)
- Fan-out SSE to multiple connected clients
- Event types: ticket_done, ticket_spawned, progress, phase_gate, failure, ram_warning
- `/api/events` endpoint — SSE stream

### 3. Middleware (internal/server/middleware.go)
- CORS (configurable origins from swarm.toml)
- Optional bearer token auth
- Request logging

### 4. CLI Command
- `cmd/serve.go` — `swarm serve [--port 8090] [--cors "*"] [--auth-token ""]`
- Starts HTTP server + watchdog in background goroutine
- Graceful shutdown on SIGINT/SIGTERM

### 5. Key Handlers
- All endpoints use standard library `net/http` (no framework)
- JSON responses with proper error codes
- SSE uses `text/event-stream` content type with `event:` + `data:` fields
- `/api/projects/:name/tickets/:id/output` — SSE stream that tails tmux pane every 2s

## Tests
Write tests FIRST:
- HTTP handler tests using httptest.NewRecorder
- Event bus fan-out (multiple subscribers)
- Auth middleware (valid/invalid/missing token)
- SSE response format

## Deliverables
- `go build ./...` passes
- `go test ./internal/server/...` passes
- Commit: `feat(sw-07): HTTP API + SSE event bus`
