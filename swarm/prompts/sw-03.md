# SW-03: Dispatcher + Phase Gates

## Context
You are building a Go CLI called `agent-swarm`. Read SPEC.md for full spec.

## Your Scope
Build the dispatcher engine in `internal/dispatcher/`.

### 1. Dispatcher (internal/dispatcher/dispatcher.go)
```go
type Signal string
const (
    SignalSpawn     Signal = ""
    SignalPhaseGate Signal = "PHASE_GATE"
    SignalAllDone   Signal = "ALL_DONE"
    SignalBlocked   Signal = "BLOCKED"
)

type Dispatcher struct {
    config  *config.Config
    tracker *tracker.Tracker
}

func New(cfg *config.Config, t *tracker.Tracker) *Dispatcher
func (d *Dispatcher) Evaluate() (Signal, []string)  // signal + spawnable ticket IDs
func (d *Dispatcher) CanSpawnMore() bool             // running < maxAgents && RAM OK
func (d *Dispatcher) NextSpawnable(n int) []string   // up to n tickets to spawn
func (d *Dispatcher) MarkDone(ticketID, sha string) (Signal, []string)  // mark done, return signal + newly unblocked
func (d *Dispatcher) MarkFailed(ticketID string) error
func (d *Dispatcher) ApprovePhaseGate() (Signal, []string)  // unlock next phase, return spawnable
func (d *Dispatcher) CurrentPhase() int
func (d *Dispatcher) PhaseStatus() PhaseStatus  // {phase, total, done, running, gateReached bool}
```

### 2. Notify Interface (internal/notify/)
- `internal/notify/notifier.go` — `Notifier` interface with `Alert` and `Info`
- `internal/notify/stdout.go` — prints to terminal with color
- Keep telegram.go as a stub (interface only, implement later)

### 3. CLI Commands
- `cmd/done.go` — `swarm done <ticket> [sha]` calls dispatcher.MarkDone
- `cmd/fail.go` — `swarm fail <ticket>` calls dispatcher.MarkFailed
- `cmd/go_cmd.go` — `swarm go [project]` calls dispatcher.ApprovePhaseGate

### Rules
- Phase gate triggers when ALL tickets in current phase are done AND next phase has tickets
- `swarm done` must: update tracker → evaluate signal → if spawnable, print what WOULD spawn (don't spawn — that's watchdog's job)
- `swarm go` must: verify phase gate is reached → approve → print spawnable tickets

## Tests
Write tests FIRST:
- Dispatcher evaluate with various tracker states
- Phase gate detection (all done in phase, mixed states)
- MarkDone chains (completing one ticket unblocks dependents)
- ApprovePhaseGate flow
