# SW-08: Integration Command

## Context
You are building a Go CLI called `agent-swarm`. Read SPEC.md for full spec.
Depends on sw-01 (tracker with DependencyOrder), sw-04 (worktree manager).

## Your Scope
Build `swarm integrate` — automated merge pass for parallel branches.

### 1. Integration Logic
```
swarm integrate [--base main] [--branch integration/v1]
```

1. Get all `done` tickets from tracker
2. Sort by `DependencyOrder()` (topological — leaves first, dependents last)
3. Create integration branch from `--base`
4. For each ticket in order:
   a. `git merge feat/<ticket> --no-ff`
   b. On conflict: stop, print which files conflict, print instructions for manual resolution
   c. On success: continue
5. After all merges: run verify command from `swarm.toml` `[integration].verify_cmd`
6. If verify passes: push integration branch
7. If `[integration].audit_ticket` is set: print that audit ticket is now spawnable

### 2. CLI Command
- `cmd/integrate.go` — `swarm integrate [--base main] [--branch integration/v1] [--dry-run]`
- `--dry-run`: print merge order and potential conflicts without executing
- Print a summary table: ticket, branch, merge result (ok/conflict/skipped)

### 3. Conflict Resolution Helper
When a conflict occurs:
```
❌ Conflict merging feat/sw-05 into integration/v1

Conflicted files:
  - internal/config/config.go
  - cmd/root.go

To resolve manually:
  cd /path/to/repo
  git merge feat/sw-05
  # fix conflicts
  git add . && git commit
  swarm integrate --continue
```

- `swarm integrate --continue` resumes from where it stopped
- Store resume state in `.swarm-integrate-state.json`

### 4. Install Command
- `cmd/install.go` — `swarm install [--user] [--interval 5m] [--uninstall]`
- Detect platform: systemd, launchd, cron
- Write appropriate service/timer/plist/cron entry per SPEC.md
- `--uninstall` removes cleanly
- Verify with dry-run after install

## Tests
Write tests FIRST:
- Merge order matches dependency graph (use temp git repo with branches)
- Conflict detection and state save
- Resume from saved state
- Install detection (mock filesystem for systemd/launchd checks)

## Deliverables
