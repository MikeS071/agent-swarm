# SW-02: Codex Tmux Backend

## Context
You are building a Go CLI called `agent-swarm`. Read SPEC.md for full spec.

## Your Scope
Implement the Codex tmux agent backend in `internal/backend/`.

### 1. Backend Interface (internal/backend/backend.go)
```go
type AgentBackend interface {
    Spawn(ctx context.Context, cfg SpawnConfig) (AgentHandle, error)
    IsAlive(handle AgentHandle) bool
    HasExited(handle AgentHandle) bool
    GetOutput(handle AgentHandle, lines int) (string, error)
    Kill(handle AgentHandle) error
    Name() string
}

type SpawnConfig struct {
    TicketID    string
    Branch      string
    WorkDir     string
    PromptFile  string
    Model       string
    Effort      string
    ExtraFlags  []string
}

type AgentHandle struct {
    SessionName string
    PID         int
    StartedAt   time.Time
}
```

### 2. Codex Implementation (internal/backend/codex.go)
- `NewCodexBackend(binary string, bypassSandbox bool) *CodexBackend`
- If `binary` is empty, auto-detect: check `codex` in PATH, then `~/.local/bin/codex`
- `Spawn`: create tmux session named `swarm-<ticketID>`, run:
  `codex exec -m <model> --dangerously-bypass-approvals-and-sandbox -C <workdir> "$(cat promptfile)"`
- `IsAlive`: `tmux has-session -t <session>` exit code 0
- `HasExited`: session exists but shell process exited (check tmux pane PID)
- `GetOutput`: `tmux capture-pane -t <session> -p -S -<lines>`
- `Kill`: `tmux kill-session -t <session>`
- List running sessions: `tmux list-sessions -F "#{session_name}"` filtered by `swarm-` prefix

### 3. Progress Parser (internal/progress/progress.go)
- `ParseMarker(output string) *Marker` — find last `PROGRESS: X/N` line
- `InferHeuristic(output string, runtime time.Duration) int` — file changes, build output, git commit detection
- `GetProgress(handle AgentHandle, backend AgentBackend, promptTasks int) TicketProgress`
- TicketProgress struct per SPEC.md
